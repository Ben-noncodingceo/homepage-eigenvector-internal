# 🚀 Cloudflare Pages 自动部署 - 简单步骤清单

> **重要说明**：此网站为**即未科技/质心教育**（Center of Mass Education Tech.）内部培训、测试、分享使用。

> **✅ 好消息**：您的 Cloudflare Pages 项目已经创建，项目名称：`homepage-eigenvector-internal`

## 📝 您需要做的事情（按顺序完成）

> 💡 **提示**：如果项目已经创建并连接了 GitHub，您可以直接跳到"第四步：配置构建设置"。

### ✅ 第一步：注册 Cloudflare 账户

1. 打开浏览器，访问：https://dash.cloudflare.com/sign-up
2. 输入您的邮箱和密码
3. 点击 **"Sign Up"**（注册）
4. 验证邮箱（检查邮箱收件箱，点击验证链接）
5. 完成注册 ✅

**预计时间**：2-3 分钟

---

### ✅ 第二步：登录并创建 Pages 项目

1. 访问：https://dash.cloudflare.com/
2. 使用刚才注册的账号登录
3. 在左侧菜单中找到 **"Workers & Pages"**，点击它
4. 点击页面上的 **"Create application"** 按钮
5. 点击 **"Pages"** 标签
6. 点击 **"Connect to Git"** 按钮 ✅

**预计时间**：1-2 分钟

---

### ✅ 第三步：连接您的 GitHub 仓库

1. 选择 **"GitHub"**（如果看到多个选项）
2. 如果第一次连接，会弹出授权页面：
   - 点击 **"Authorize Cloudflare Pages"**
   - 选择您的仓库（可以选择单个仓库或所有仓库）
   - 点击 **"Install"** 完成授权
3. 在仓库列表中找到并选择：`homepage-eigenvector-internal`（或您的仓库名称）
4. 点击 **"Begin setup"** ✅

**预计时间**：2-3 分钟

---

### ✅ 第四步：配置构建设置（重要！）

在设置页面，按照以下内容填写：

#### 1. 项目名称
- **Project name**: 输入 `homepage-eigenvector-internal`（或您喜欢的名称）
- 这个名称会成为网站地址的一部分

#### 2. 生产分支
- **Production branch**: 选择 `master`（如果您的仓库主分支是 `master`）
- 或者选择 `main`（如果您的仓库主分支是 `main`）

#### 3. 构建设置（点击 "Show advanced settings" 展开）
- **Framework preset**: 选择 `Jekyll`（在下拉菜单中）
- **Build command**: 输入 `bundle exec jekyll build`
- **Build output directory**: 输入 `_site`
- **Root directory**: 留空（不要填任何东西）

#### 4. 环境变量（可选，但推荐添加）
点击 **"Add environment variable"** 添加：

**第一个变量：**
- **Variable name**: `JEKYLL_ENV`
- **Value**: `production`

**第二个变量：**
- **Variable name**: `RUBY_VERSION`  
- **Value**: `3.1`

完成后，点击页面底部的 **"Save and Deploy"** 按钮 ✅

**预计时间**：3-5 分钟

---

### ✅ 第五步：等待部署完成

1. 点击 **"Save and Deploy"** 后，页面会显示构建进度
2. 等待 3-5 分钟（首次构建需要较长时间）
3. 看到绿色的 ✅ 勾号表示部署成功
4. 页面会显示您的网站地址，格式类似：`https://homepage-eigenvector-internal.pages.dev` ✅

**预计时间**：3-5 分钟

---

### ✅ 第六步：测试您的网站

1. 点击 Cloudflare 提供的网站地址（`.pages.dev` 结尾的链接）
2. 检查网站是否正常显示
3. 测试右上角的语言切换功能
4. 如果一切正常，恭喜您！部署成功！🎉

**预计时间**：1-2 分钟

---

## 🎯 完成！现在您的网站已经自动部署了

### 以后如何更新网站？

非常简单！只需要：

1. 在您的代码中做任何修改
2. 将修改推送到 GitHub 的 `master` 或 `main` 分支
3. Cloudflare 会自动检测到更改并重新部署
4. 等待 1-2 分钟，网站就会自动更新！

**您不需要做任何其他操作！** ✨

## ❓ 常见问题

### Q: 构建失败了怎么办？
**A**: 
1. 在 Cloudflare Dashboard 中点击失败的部署
2. 查看构建日志中的错误信息
3. 检查是否按照步骤四正确配置了所有设置
4. 如果还是失败，可以联系技术支持

### Q: 网站地址可以改吗？
**A**: 
- 项目名称可以改，但需要重新部署
- 或者您可以添加自定义域名（需要您有自己的域名）

### Q: 部署需要多长时间？
**A**: 
- 首次部署：3-5 分钟
- 后续更新：1-2 分钟

### Q: 需要付费吗？
**A**: 
- Cloudflare Pages 有免费套餐，对于个人网站完全够用
- 免费套餐包括：500 次构建/月，无限带宽
